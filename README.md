# Simple RBN Spotter

**Simple RBN Spotter** — минималистичный веб-монитор для Reverse Beacon Network (RBN)

Показывает, кого **вы** услышали как споттер.

---

## Возможности

- **Поиск по позывному споттера** — введите позывной (например, `R4NCU`) и увидите все споты, сделанные этим споттером за последний час.
- **Автообновление каждые 5 секунд** — таблица и карта обновляются в реальном времени.
- **Интерактивная карта (Leaflet)** — тёмная/светлая тема, позывной споттера отмечен жёлтой точкой, услышанные станции — цветными маркерами по диапазонам (160M, 80M, 40M…), линии от споттера к каждой станции, popup-окна с деталями.
- **Таблица спотов** — колонки: Heard (+флаг страны), расстояние (км), SNR (дБ), время (UTC), частота (кГц). Цветовая индикация SNR (зелёный/жёлтый/оранжевый). Сортировка кликом по заголовкам. Подсветка новых спотов.
- **Флаги стран** — определяются по префиксу позывного через встроенный словарь (~80+ стран).
- **Тёмная/светлая тема** — переключатель, сохраняется в localStorage.
- **Адаптивный дизайн** — работает на десктопе и мобильных устройствах.
- **Автосохранение позывного** — последний запрос сохраняется в localStorage.

---

## Отличие от Simple RBN Monitor

| | Simple RBN Monitor (Spotted) | Simple RBN Spotter |
|---|---|---|
| **Показывает** | Кто услышал **вас** | Кого **вы** услышали |
| **API параметр** | `dx=CALL` | `de=CALL` |
| **Колонка таблицы** | Spotter (кто споткнул) | Heard (кого услышали) |

---

## Сильные стороны

1. **Минимализм** — всего 2 файла кода (server.py + spotter.html), ноль внешних зависимостей.
2. **Простота** — делает одну вещь и делает её хорошо: показать, кого слышит твой споттер прямо сейчас.
3. **Скорость** — встроенное кэширование (3 сек), переиспользование HTTP-соединений, автообновление 5 сек.
4. **Самодостаточность** — фронтенд в одном HTML-файле (CSS + JS + HTML внутри), всё включено.

---

## Размещение на Render.com (free tier)

Приложение можно запустить на **Render.com** — бесплатный хостинг (Render Web Service).

### Особенности бесплатного тарифа

- **Сервер «засыпает»** после ~15 минут бездействия.
- **При первом обращении** после сна требуется **30–60 секунд на «пробуждение»**.
- После пробуждения всё работает штатно, автообновление 5 сек.
